General Rules

* Keep responses concise.
* Prefer commands over long explanations.
* Think before acting.
* Explain briefly what you are going to do before executing commands.
* Never make destructive changes without explicit confirmation unless explicitly requested.
* Favor stable and maintainable solutions over clever ones.

CLI Preferences

* Prefer rg over grep if available.
* Prefer fd over find if available.
* Use jq for JSON processing.
* Use yq for YAML processing.
* Prefer bat over cat if available.
* Prefer eza over ls if available.
* Prefer dust over du if available.
